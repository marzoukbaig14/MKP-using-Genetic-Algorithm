Generating Function-Informed Initialization for Genetic Algorithms on the Multidimensional Knapsack Problem

Muhammad Marzouk Baig  -  MS AI Candidate, Northeastern University, Portland ME  -  baig.muham@northeastern.edu
The Problem

Standard GA initialization samples each bit at 50/50, ignoring constraint structure. On tight instances (α = 0.25), every initial solution is infeasible - selection has nothing to work with. The algorithm never starts.

Our Approach
1
For each item j, compute the sub-generating function over all other items - enumerates all 2n-1 possible completions.
gj(x) = ∏k≠j (1 + xwₖ)
2
Count what fraction of completions stay feasible when item j is included (ρ¹_j) vs. excluded (ρ⁰_j).
3
Set per-item sampling probability:
p_j = ρ¹_j / (ρ¹_j + ρ⁰_j)
- p_j = 0.5 when instance is loose (auto-degenerates to uniform)
- p_j → 0 when item blocks all feasible completions
4
Initialize each solution: set bit j = 1 with probability p_j.
Scalability
★
GPU Monte Carlo - draw M = 10⁷ random binary vectors, check feasibility in parallel.
SE ≤ 0.02% at M = 10⁷
Runtime: 5-8 sec (T4 GPU)
n=100, m=3, M=10,000,000

One-time precomputation. No objective function required. No LP relaxation. Purely constraint-driven.

Algorithmic complexity
Exact:     O(n · 2n · m) — intractable
MC approx: O(n · M · m) — linear, GPU-parallel
Approximation trades exponential enumeration
for controllable error: SE ≤ 1/(2√M)

At M=10&sup7;: SE ≤ 0.016%. Memory: ~125 MB for M×(n−1) sample matrix. Precomputation is paid once per instance — cost per initialized solution decreases with population size.

Self-Calibrating Property

At α = 0.75, every item has pj = 0.5. The framework automatically degenerates to uniform Chu & Beasley initialization when the instance is loose - correct behavior by theory, confirmed by experiment.

The standard repair operator computes uj = vj / Σ(wji/Wi). It requires objective values. It fails silently on three common problem classes.

Where Repair Fails - Our Method Works
No objective (CSP). Efficiency scores can't be computed. GF initialization needs only weights and capacities.
Noisy objectives. When vj values are uncertain, the efficiency ranking is unreliable. GF initialization ignores objective values entirely.
Adversarial correlation. Negatively correlated instances cause the greedy repair to systematically drop high-value items. GF initialization is unaffected.
Part 2 (in progress): demonstrating repair failure on adversarial instances following Hill et al. instance generation protocol. Guided by Prof. Raymond Hill (AFIT / Wright State).
Mathematical Novelty

Sub-generating functions correspond exactly to order-1 schemas in Holland's framework. Each gj enumerates all 2n-1 strings that agree with item j's fixed value. First application of generating functions to bias GA initialization in the EC literature.

Requires n evaluations (not 3n). Recovers uniform as a special case. Interpretable and theoretically principled.
